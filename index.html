How MoodleGPT 3 answers in milliseconds

Each quiz interaction follows a deterministic pipeline that prioritises local data, preserves your privacy, and only touches Gemini when you have explicitly supplied a key.

01

Select any Moodle question

The content script recognises Moodle question containers, normalizes the text, and extracts answer options--even tricky drop-down blanks. The same extraction powers manual clicks and automation.

02

Local answer pack check

If a match is found inside the bundled offline answer pack (321 curated entries), the answer is surfaced immediately and can be written back to Google Sheets for historical review.

03

Gemini fallback with your key

When no local match exists and the fallback toggle is enabled, MoodleGPT 3 calls Gemini 2.5 via the API key you store in Chrome storage. Responses are tightened to exact option text before selection.

04

Optional logging & sync

All successful answers can be mirrored to your Google Apps Script endpoint for auditing. Enhanced logging keeps the last 100 errors locally so support can reproduce issues quickly.

Automation paced like a human

Automation is disabled on installation. Once you opt in, MoodleGPT 3 sticks to your window, tracks every answer, and refuses to rush the attempt.

0 s

Automation detects an unanswered true/false or gap-select question.

+4 s

Minimum delay respected. A local match is preferred; Gemini fallback kicks in only if allowed.

+8 s

Answer is selected, the overlay updates, and your status widget stays semi-transparent.

+15 s

Maximum default delay reached. MoodleGPT 3 advances the quiz and waits for the next unanswered block.

You can raise the delay ceiling up to 300 s and lower the floor to 1 s from the settings screen. Every change propagates instantly via chrome.storage sync.

Privacy, logging, and support

Extension diagnostics only move when you request them. You retain full control over the data MoodleGPT 3 sees and stores.

Local-first storage

Gemini keys, automation ranges, and language preferences reside in Chrome storage. Clearing the extension data wipes everything.

Error insights on demand

The error handler caps logs at 100 entries and exposes a downloadable JSON report so support can reproduce issues without remote logging.

Google Sheets export

The background script ships with an Apps Script webhook. Keep it enabled for history, or comment it out if you prefer zero external requests.
